# Система инвайтов CorpMeet — документация для бота

## Типы ссылок-приглашений

В системе два независимых типа ссылок. Оба приходят через `/start` в боте.

| Тип | Префикс токена | Использование | Срок действия |
|-----|---------------|--------------|---------------|
| Персональная | `invite_TOKEN` | Одноразовая. Владелец/админ генерирует для одного человека | Гасится после первого использования |
| Публичная | `ws_CODE` | Многоразовая. Можно публиковать в чатах, рассылках | Действует пока владелец не обновит код пространства |

---

## 1. Персональная ссылка (`invite_TOKEN`)

### Как генерируется
Владелец/админ нажимает "Создать ссылку-приглашение" в настройках пространства.  
Бэкенд создаёт запись `WorkspaceMember` (без привязки к конкретному пользователю) и возвращает ссылку:
```
https://t.me/{BOT_USERNAME}?start=invite_{token}
```

### Что делает бот при `/start invite_{TOKEN}`

**Шаг 1 — Убедиться что токен существует:**
```
GET /api/v1/internal/invites/by-token/{token}
```
Ответ:
```json
{
  "member_id": 42,
  "workspace_id": 7,
  "workspace_name": "Моя компания",
  "status": "pending",
  "inviter_name": "Иван Иванов",
  "user": null
}
```
- Если `user: null` — это анонимный инвайт (новый тип). Переходи к шагу 2.
- Если `user` заполнен — это старый инвайт по username. Переходи к шагу 3 (показать кнопки Accept/Reject).
- Если 404 — ссылка недействительна или уже использована.

**Шаг 2 — Клейм анонимного инвайта (auto-accept):**
```
POST /api/v1/internal/invites/claim
```
Body:
```json
{
  "token": "TOKEN",
  "telegram_id": 123456789,
  "username": "username",
  "first_name": "Имя",
  "last_name": "Фамилия"
}
```
Ответ:
```json
{
  "ok": true,
  "already_member": false,
  "status": "active",
  "workspace_name": "Моя компания"
}
```
- `already_member: true` — пользователь уже состоит в пространстве (инвайт всё равно погашен).
- После успеха показать сообщение и **WebApp-кнопку** (не обычную URL-ссылку!):
  ```
  ✅ Вы добавлены в пространство «{workspace_name}»!
  [Открыть мини-приложение]  ← InlineKeyboardButton с web_app=WebAppInfo(url=f"{WEBAPP_URL}?invite_token={token}")
  ```
  Именно WebApp-кнопка передаёт Telegram-авторизацию автоматически. Обычная URL-ссылка откроет браузер без сессии — пользователь увидит экран регистрации вместо пространства.
  Токен передаётся как URL query-параметр `invite_token`. Мини-приложение автоматически читает его и подключает пользователя к пространству при входе (для новых пользователей — при регистрации, для существующих — при логине).

> ⚠️ **Важно:** вызов `POST /invites/{member_id}/accept` для анонимного инвайта (где `user: null`) вернёт `400 Bad Request`. Для анонимных инвайтов используется **только** `/claim`. `/accept` работает только для инвайтов с заполненным `user`.

**Шаг 3 — Инвайт по username (если `user` не null, старый флоу):**  
Показать кнопки "Принять / Отказаться".
- Принять: `POST /api/v1/internal/invites/{member_id}/accept`
- Отказаться: `POST /api/v1/internal/invites/{member_id}/reject`

### Важно
- Если пользователь ещё не зарегистрирован в системе — `/claim` создаёт его автоматически.
- Токен одноразовый: после `/claim` обнуляется, повторный переход по ссылке даст 404.

## 2. Публичная ссылка (`ws_CODE`)

### Как генерируется
Автоматически для каждого пространства. Владелец видит её в "Настройки → Общее → Публичная ссылка Telegram".  
Формат:
```
https://t.me/{BOT_USERNAME}?start=ws_{invite_code}
```
Владелец может обновить `invite_code` — старая ссылка перестаёт работать немедленно.

### Что делает бот при `/start ws_{CODE}`

Один запрос:
```
POST /api/v1/internal/workspaces/join-by-invite
```
Body:
```json
{
  "invite_code": "CODE",
  "telegram_id": 123456789,
  "username": "username",
  "first_name": "Имя",
  "last_name": "Фамилия"
}
```
Ответ:
```json
{
  "ok": true,
  "already_member": false,
  "status": "active",
  "workspace_name": "Моя компания"
}
```
- `already_member: true` — уже состоит, ничего не меняется.
- Пользователь создаётся автоматически если не зарегистрирован.
- После успеха показать сообщение и **WebApp-кнопку** (не обычную URL-ссылку!):
  ```
  ✅ Вы вступили в пространство «{workspace_name}»!
  [Открыть мини-приложение]  ← InlineKeyboardButton с web_app=WebAppInfo(url=f"{WEBAPP_URL}?ws_code={code}")
  ```
  Код передаётся как URL query-параметр `ws_code`. Мини-приложение автоматически читает его и добавляет пользователя в пространство при входе (для новых пользователей — при регистрации, для существующих — при логине).
- Если 404 — код не найден (устарел после обновления или пространство не существует).

## 3. Браузерная сессия (существующий флоу, без изменений)

Если токен не начинается на `invite_` и не начинается на `ws_` — это QR/браузерная авторизация:
```
POST /api/v1/internal/auth/consume-session
```

## Алгоритм определения типа в `/start`

```python
token = command.args  # аргумент после /start

if not token:
    # обычный /start — ensure_user, показать приветствие
    ...
elif token.startswith("invite_"):
    raw = token[len("invite_"):]
    # персональный инвайт
    ...
elif token.startswith("ws_"):
    code = token[len("ws_"):]
    # публичный инвайт пространства
    ...
else:
    # браузерная сессия (QR-авторизация)
    ...
```

## Регистрация нового пользователя

Все три флоу (`/claim`, `join-by-invite`, `consume-session`) создают пользователя автоматически если `telegram_id` не найден в базе. Отдельно вызывать `/internal/users/ensure` не нужно — это уже встроено в каждый endpoint.

После успешного добавления в пространство рекомендуется проверить `has_position` из ответа (если endpoint его возвращает) и предложить пользователю выбрать должность если она не задана.
